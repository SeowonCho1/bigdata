표준 ML 파이프라인 (7단계)

교재 모든 챕터가 동일한 7단계 흐름을 따릅니다.

1. 문제정의→ 2. 데이터 불러오기→ 3. EDA→ 4. 전처리→ 5. 검증 분할→ 6. 학습·평가→ 7. 예측·제출
이론. 작업형2는 지도학습(Supervised Learning) 문제입니다. train 데이터로 입력(X)과 정답(y)의 관계를 학습하고, test 데이터의 정답을 예측해 result.csv로 제출합니다. 핵심은 train에서 한 전처리를 test에 동일하게 적용하는 것입니다(데이터 누수 방지).

고정 패턴 (모든 챕터 공통)

  • 분할: train_test_split(..., test_size=0.2, random_state=0)
  • 제출: pd.DataFrame({'pred': ...}).to_csv("result.csv", index=False)
  • 주력 모델: RandomForest 계열, 보조로 LightGBM

문제 유형 & 제출 형식

유형별로 모델·평가지표·제출 컬럼이 달라집니다 — 가장 헷갈리는 부분.

유형평가지표pred 컬럼 내용예측 함수
이진분류 (확률)ROC-AUC양성 클래스 확률predict_proba(test)[:, 1]
이진분류 (라벨)F1예측 클래스 (0/1)predict(test)
다중분류accuracy / f1-macro예측 클래스 라벨predict(test)
회귀RMSE / R² / RMSLE예측 연속값predict(test)
주의. ROC-AUC 문제는 반드시 predict_proba(test)[:, 1](양성 확률)을 제출해야 합니다. predict()(0/1 라벨)를 내면 점수가 크게 떨어집니다. 인덱스 [:, 1]은 model.classes_의 두 번째 클래스 확률입니다.

전처리 이론 & 코드

결측치 → 이상치 → 인코딩 → 스케일링 순서로 처리.

① 결측치 (Missing Values)

이론. 모델은 보통 결측치(NaN)를 처리하지 못합니다(LightGBM 제외). 행을 지우거나 적절한 값으로 채웁니다. test 행은 절대 삭제 금지 — 모든 test 행에 예측값이 있어야 채점됩니다. train 행은 삭제 가능.
방법코드적합한 경우
행 삭제train.dropna(subset=['col'])train에서 결측 적을 때
열 삭제df.drop(['col'], axis=1)결측이 너무 많은 컬럼
최빈값(범주형)df['c'].fillna(df['c'].mode()[0])범주형 컬럼
새 범주 "X"df['c'].fillna('X')결측 자체가 의미 있을 때
평균/중앙값(수치)df['n'].fillna(df['n'].median())수치형 컬럼
최솟값/-1df['n'].fillna(-1)트리 모델 구분용

② 이상치 (Outliers)

train = train[train['age'] > 0]   # 음수 나이 등 비정상값 제거 (train만!)

③ 인코딩 (Encoding) — 범주형 → 숫자

이론. 모델은 문자열을 못 받습니다. 범주형을 숫자로 바꿔야 합니다.
  • 원-핫 인코딩: 각 범주를 0/1 컬럼으로. 순서 없는 명목형에 적합.
  • 레이블 인코딩: 범주를 0,1,2…정수로. 트리 모델에 무난.

원-핫 인코딩

train = pd.get_dummies(train)
test  = pd.get_dummies(test)
# 컬럼 수 불일치 방지 (필수!)
train, test = train.align(
    test, join='left', axis=1, fill_value=0)

레이블 인코딩

from sklearn.preprocessing import LabelEncoder
cols = train.select_dtypes('object').columns
for col in cols:
    le = LabelEncoder()
    train[col] = le.fit_transform(train[col])
    test[col]  = le.transform(test[col])
함정. get_dummies를 train/test 따로 돌리면 범주가 달라 컬럼 수가 어긋납니다(ch2 사례: 107 vs 103). concat 후 인코딩하거나 align으로 맞추세요. LightGBM은 astype('category')로 인코딩 없이 처리 가능(ch5).

④ 스케일링 (Scaling) — 수치형 범위 통일

이론. 변수마다 단위/범위가 다르면 거리·계수 기반 모델이 왜곡됩니다. 트리 모델(RF, LGBM)은 스케일링 영향이 적지만, 선형/거리 모델엔 중요합니다.
스케일러변환특징
MinMaxScaler0~1 범위최소-최대 정규화
StandardScaler평균0, 표준편차1Z-점수 표준화
RobustScaler중앙값/IQR 기준이상치에 강건
 
scaler = StandardScaler()
train[num_cols] = scaler.fit_transform(train[num_cols])
test[num_cols]  = scaler.transform(test[num_cols])   # test는 transform만!
핵심 원칙. 스케일러·인코더는 fit_transform은 train에만, test에는 transform만 적용합니다. test 정보로 fit하면 데이터 누수입니다.

모델 종류 & 선택

시험에서는 트리 앙상블(RF, LightGBM)이 표준. 베이스라인 후 하이퍼파라미터 튜닝.

모델분류회귀특징
랜덤포레스트RandomForestClassifierRandomForestRegressor안정적 베이스라인, 스케일링 불필요
LightGBMLGBMClassifierLGBMRegressor빠르고 강력, 범주형 native 지원
선형회귀—LinearRegression회귀 기본 비교용 (ch4)

자주 쓰는 하이퍼파라미터

  • random_state=0 — 재현성 (항상 고정)
  • n_estimators=200~500 — 트리 개수 (많을수록 안정, 느림)
  • max_depth=5~20 — 트리 깊이 (과적합 제어)
  • class_weight='balanced' — 불균형 분류 대응
  • verbose=-1 — LightGBM 로그 끄기

교차검증 (ch7)

평가지표 (ch3 핵심)

문제 유형별로 어떤 지표를 쓰는지, 각 지표가 무엇을 측정하는지.

분류 — 혼동행렬 기반

이론. 혼동행렬(Confusion Matrix): TP(맞춘 양성), FP(틀린 양성), FN(놓친 양성), TN(맞춘 음성).
  • 정밀도(Precision) = TP/(TP+FP) — "양성이라 한 것 중 진짜 비율"
  • 재현율(Recall) = TP/(TP+FN) — "실제 양성 중 찾아낸 비율"
  • F1 = 정밀도·재현율의 조화평균 = 2PR/(P+R)
함수측정주요 인자
accuracy_score전체 정확도 = 맞은 수/전체—
precision_score정밀도pos_label, average
recall_score재현율(민감도)pos_label, average
f1_scoreF1 점수pos_label, average='macro'
roc_auc_scoreROC 곡선 아래 면적 (확률 순위 품질)확률 입력 필요
문자열 라벨 / 다중분류 주의. 이진 문자열은 pos_label='>50K' 지정. 다중분류는 average='macro'(클래스 균등) 또는 'weighted'(빈도 가중)를 씁니다. ROC-AUC는 항상 확률 입력.

회귀

함수측정특징
mean_squared_errorMSE = 평균((y−ŷ)²)큰 오차에 민감
root_mean_squared_errorRMSE = √MSE타겟과 같은 단위 ← 가장 흔함
mean_absolute_errorMAE = 평균(|y−ŷ|)이상치에 강건
r2_scoreR² (설명된 분산 비율)1에 가까울수록 좋음
root_mean_squared_log_errorRMSLE = √MSLE가격·개수 데이터, 과소예측 패널티
R² 이론. R² = 1 − (SS_res / SS_tot). 모델이 평균선 대비 얼마나 잘 설명하는지 비율. 0이면 평균만큼, 1이면 완벽, 음수면 평균보다 나쁨.

제출 코드 치트시트

이 한 장이면 작업형2는 끝. 유형에 맞게 모델·지표·제출만 교체.

import pandas as pd
from sklearn.model_selection import train_test_split

train = pd.read_csv("train.csv")
test  = pd.read_csv("test.csv")

# 1) 타겟 분리
target = train.pop("타겟")

# 2) 전처리 (인코딩)
train = pd.get_dummies(train); test = pd.get_dummies(test)
train, test = train.align(test, join='left', axis=1, fill_value=0)

# 3) 검증 분할
X_tr, X_val, y_tr, y_val = train_test_split(train, target, test_size=0.2, random_state=0)

# 4) 모델 학습 (유형별 택1)
import lightgbm as lgb
from sklearn.metrics import f1_score, roc_auc_score, root_mean_squared_error

# --- 분류(라벨) ---
model = lgb.LGBMClassifier(random_state=0, verbose=-1)
model.fit(X_tr, y_tr)
print(f1_score(y_val, model.predict(X_val), average='macro'))

# --- 이진분류(확률) ---
# print(roc_auc_score(y_val, model.predict_proba(X_val)[:,1]))

# --- 회귀 ---

# model.fit(X_tr, y_tr)
# print(root_mean_squared_error(y_val, model.predict(X_val)))

# 5) 예측 & 제출
pred = model.predict(test)              # 분류 라벨 / 회귀값
# pred = model.predict_proba(test)[:,1] # 이진 확률
